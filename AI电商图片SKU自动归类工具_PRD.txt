AI电商图片SKU自动归类工具 - 产品需求文档（PRD）


一、项目背景

团队使用AI生成大量电商图片，图片自动保存至Mac「下载」文件夹。每张SKU约对应5张图片，日均处理量可达数百张。目前纯靠人工手动分类、重命名、整理到SKU文件夹，效率低且易出错。

核心目标：提供一个极简Mac桌面应用（.app格式），实现「输入货号 → 拖拽图片 → 自动建文件夹 + 重命名 + 归档」。

扩展目标：应用内建立标准的数据索引层，向外暴露API接口，供抖店、ERP等外部系统通过货号自动匹配图片并调用上传。


二、技术方案

- 平台：macOS桌面应用（.app格式）
- 技术栈建议：Electron（前端）+ Node.js（主进程）或 Python GUI + FastAPI/Flask
- 本地数据库：SQLite（轻量、零配置，内嵌在应用中）
- 本地服务：应用启动后自动在后台启动HTTP服务（端口可配置，默认8765），对外提供REST API
- 签名需求：第一版为内部自用，无需Apple Developer付费账号


三、功能需求

3.1 新建与归类（核心流程）

步骤1：打开应用，点击「新建」
步骤2：输入SKU货号（校验非空；若同名文件夹已存在，提示"追加"或"取消"）
步骤3：拖拽或点击上传图片到编辑区
步骤4：点击「完成」
  → 系统行为：在用户指定的根目录下创建以货号命名的文件夹，将图片移入并重命名，同时写入数据库索引

数据库同步字段：
- 货号（SKU）
- 文件夹绝对路径
- 图片文件名列表及对应顺序
- 创建时间（精确到秒）

3.2 命名规范化

- 命名规则：货号-序号.原扩展名
  示例：A1001-1.jpg、A1001-2.png……
- 追加逻辑：向已有文件夹追加图片时，自动检测当前最大序号，顺延递增，绝不覆盖已有文件

3.3 搜索与编辑模块

功能1：按货号搜索
  - 输入货号，检索并定位到对应的文件夹路径

功能2：按日期搜索（新增）
  - 支持按创建日期筛选
  - 例如输入"今天"或具体日期（如2025-05-14），列出当日创建的所有SKU文件夹
  - 搜索框旁增加日期筛选器（快捷选项：今天 / 昨天 / 近7天 / 自定义日期）

功能3：按日期导出（新增）
  - 按日期筛选结果后（如搜索5月13号，结果有20个文件夹），支持一键导出
  - 导出位置由用户指定（弹出文件夹选择器）
  - 导出格式：将该日期下的所有SKU文件夹整体打包成一个压缩包（.zip格式）
  - 压缩包命名建议：SKU导出_2025-05-13.zip

功能4：缩略图预览
  - 点击进入文件夹后，以缩略图网格/列表形式展示该SKU下的所有图片

功能5：删除同步
  - 在应用内删除某张图片，物理文件同步从文件夹中移除，数据库索引同步删除

3.4 数据接口/API模块

设计原则：
- 应用启动后，后台自动启动本地HTTP服务（端口可配置，默认8765）
- 所有图片和货号信息通过数据库索引管理，外部系统通过API调用即可

接口规范：
基础地址：http://localhost:8765/api/v1

接口1：获取货号列表
  方法：GET
  路径：/skus
  参数：date（可选，如2025-05-14）
  返回：所有货号列表（含创建时间、图片数量），支持按日期筛选

接口2：获取图片列表
  方法：GET
  路径：/skus/{货号}/images
  返回：该货号下的所有图片文件名、序号、URL

接口3：获取单张图片
  方法：GET
  路径：/skus/{货号}/images/{文件名}
  返回：直接返回图片二进制流（Content-Type: image/*）

接口4：货号是否存在
  方法：GET
  路径：/skus/{货号}/exists
  返回：布尔值，用于上传前校验

图片访问URL示例：
http://localhost:8765/api/v1/skus/A1001/images/A1001-1.jpg

调用方对接说明：
- 抖店/抖音开放平台：可通过上述HTTP URL直接读取图片进行上传
- ERP系统：调用 /skus/{货号}/images 获取图片列表，再遍历获取每张图片的二进制数据或本地路径进行上传
- 鉴权（第一版）：内部局域网使用，暂时不做复杂鉴权，后续如需对外暴露再增加Token或IP白名单


四、数据与架构设计

4.1 数据库设计（SQLite）

表：sku_index

| 字段        | 类型      | 说明                     |
|-------------|-----------|--------------------------|
| id          | INTEGER   | 主键，自增               |
| sku_no      | TEXT      | 货号，唯一索引           |
| folder_path | TEXT      | 文件夹绝对路径           |
| image_count | INTEGER   | 当前图片数量             |
| created_at  | TIMESTAMP | 文件夹创建时间，精确到秒 |
| updated_at  | TIMESTAMP | 最后更新时间             |

表：sku_images

| 字段      | 类型    | 说明                 |
|-----------|---------|----------------------|
| id        | INTEGER | 主键，自增           |
| sku_no    | TEXT    | 货号，关联sku_index  |
| file_name | TEXT    | 文件名（如A1001-1.jpg）|
| seq_no    | INTEGER | 序号（1,2,3...）     |
| file_path | TEXT    | 文件绝对路径         |

五、全局配置

- 归档根目录：用户自定义图片归档根目录，应用记住该路径
- API端口：默认8765，用户可在设置中修改
- 开机自启：可选，应用启动时自动开启API服务


六、非功能需求

- 性能：单次处理图片量 ≤ 50张，秒级响应；API响应 < 200ms
- 并发：单SKU操作，不支持批量多SKU同时创建（降低操作错误率）
- 兼容性：接口返回标准JSON，图片流支持常见格式（jpg/png/webp）


七、待开发确认（给开发者）

1. 技术栈最终选型（Electron + Node.js 后端 or Python + 轻量GUI）
2. API鉴权策略（第一版是否先做无鉴权内网版）
3. 是否需要在UI中展示API服务运行状态（如"服务已启动，端口8765"）
